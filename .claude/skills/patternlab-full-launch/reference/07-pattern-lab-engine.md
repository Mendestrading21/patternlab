# 07 — Laboratoire de patterns

## Objectif

Construire un moteur de reconnaissance et d’analyse de patterns, pas une encyclopédie statique.

## Modèle Pattern

- id
- slug
- name
- aliases
- family
- direction
- difficulty
- prerequisites
- definition
- recognitionRules
- invalidationRules
- commonMistakes
- confirmationSignals
- examples
- counterExamples
- exercises
- sources
- legacy metadata
- review status

## Familles initiales

- chandeliers ;
- supports/résistances ;
- tendances ;
- triangles ;
- drapeaux/fanions ;
- doubles sommets/creux ;
- tête-épaules ;
- biseaux ;
- canaux ;
- gaps ;
- breakouts/faux breakouts ;
- divergences.

## Interaction

- sélectionner ;
- tracer ;
- déplacer ;
- zoomer ;
- confirmer ;
- comparer ;
- identifier l’invalidation.

## Progression

MVP : 20 à 30 patterns.

Architecture : extensible vers 500+ concepts sans changer le moteur.

## Données

Utiliser :

- données synthétiques reproductibles ;
- cas pédagogiques contrôlés ;
- exemples réels sous droits/licences vérifiés ;
- provenance et date.
